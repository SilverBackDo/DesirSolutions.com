# Repository Docs Index

Use `docs/` for repo documentation that supports the platform but is not itself a runtime entry point.

## Sections

- `ai-factory/` - architecture notes, autonomy posture, and multi-agent design references
- `contracts/` - trust-package contract templates and addendum summaries
- `n8n/` - deployment, PostgreSQL, and CRM integration SOPs for the n8n stack
- `operations/` - operator runbooks and operational procedures
- `security/` - security overview, incident policy, secret rotation, and subprocessors
- `archive/south-dakota/` - archived South Dakota entity research kept for reference only

## Root-Level Files That Stay At The Top

- `README.md` - repo entry point
- `TRUST_PACKAGE.md` - buyer/procurement trust summary
- website entry files and deployment assets

If a Markdown file does not need to be a public entry point or runtime artifact, it should live under `docs/`.
